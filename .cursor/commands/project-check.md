# 명령 개요

이 문서는 프로젝트의 상태를 확인하고 관리하는 프로젝트 매니저 역할을 수행하는 명령입니다. 프로젝트의 진척도, 우선순위, 완료 여부를 체계적으로 분석하고 문서를 갱신합니다.

## 역할

이 명령을 실행하면 AI는 프로젝트 매니저로서 다음 작업을 수행합니다:

1. 프로젝트 전체 상태 파악
2. API와 UI 구현 상태 비교 분석
3. 진행률 계산 및 우선순위 재평가
4. 다음 단계 제안
5. 분석 결과 문서 갱신

## 참조 파일

이 명령을 수행할 때는 반드시 다음 네 가지 파일을 참조해야 합니다:

### 1. API 참조 파일

- `@.cursor/commands/reference-api-files.md`
- **용도**: API 엔드포인트 구조와 파일 경로 확인
- **확인 항목**: 컨트롤러, 서비스, 리포, 스키마, 타입 파일 존재 여부

### 2. UI 참조 파일

- `@.cursor/commands/reference-ui-files.md`
- **용도**: UI 컴포넌트 및 페이지 구조 확인
- **확인 항목**: 컴포넌트, 엔티티, 스키마, 타입 파일 경로

### 3. 관리자 우선순위 분석 문서

- `@md/admin-priority-analysis.md`
- **용도**: 관리자 기능의 우선순위 및 구현 상태 확인
- **갱신 대상**: 이 파일을 분석 결과로 갱신

### 4. UI 컴포넌트 우선순위 문서

- `@md/ui-components-priority.md`
- **용도**: UI 컴포넌트의 우선순위 및 구현 상태 확인
- **갱신 대상**: 이 파일을 분석 결과로 갱신

## 체크 프로세스

### 1단계: 파일 존재 여부 확인

#### API 파일 체크

`reference-api-files.md`에 나열된 모든 API 관련 파일의 존재 여부를 확인합니다:

- 컨트롤러 파일 존재 여부
- 서비스 파일 존재 여부
- 리포 파일 존재 여부
- OpenAPI 문서 파일 존재 여부
- 스키마 파일 존재 여부
- 타입 파일 존재 여부

각 엔드포인트 그룹별로:

- ✅ 완전히 구현됨
- ⚠️ 부분적으로 구현됨
- ❌ 미구현

#### UI 파일 체크

`reference-ui-files.md`에 나열된 경로를 기반으로 실제 파일 존재 여부를 확인합니다:

- 컴포넌트 파일 존재 여부
- 페이지 라우트 파일 존재 여부
- 엔티티 파일 존재 여부
- 스키마 파일 존재 여부
- 타입 파일 존재 여부

각 컴포넌트별로:

- ✅ 완료
- ⚠️ 부분 완료 (컴포넌트는 있으나 페이지 미연결 등)
- ❌ 미구현

### 2단계: API-UI 매칭 분석

각 API 엔드포인트에 대응하는 UI 컴포넌트가 있는지 확인합니다:

- API가 존재하지만 UI가 없는 경우
- UI가 존재하지만 API가 없는 경우
- 둘 다 존재하지만 연결되지 않은 경우

### 3단계: 진행률 계산

#### 전체 진행률

- 총 필수 컴포넌트 수
- 완료된 컴포넌트 수
- 진행률 백분율

#### 카테고리별 진행률

- 인증 기능 진행률
- 사용자 기능 진행률
- 포스트 기능 진행률
- 댓글 기능 진행률
- 카테고리 기능 진행률
- 태그 기능 진행률
- 구독 기능 진행률
- 관리자 기능 진행률

#### 우선순위별 진행률

- 우선순위 매우 높음 진행률
- 우선순위 높음 진행률
- 우선순위 중간 진행률
- 우선순위 낮음 진행률

### 4단계: 우선순위 재평가

현재 구현 상태를 바탕으로 우선순위를 재평가합니다:

#### 우선순위 결정 기준

1. **블로그 핵심 기능**: 포스트 목록/상세, 댓글 시스템
2. **사용자 경험**: 프로필 관리, 인증 기능
3. **탐색 기능**: 카테고리, 태그 탐색
4. **구독 기능**: 카테고리/태그 구독
5. **관리자 기능**: 관리자 대시보드, 사용자 관리

#### 우선순위 조정 고려사항

- API 완성도가 높은데 UI가 없는 경우 → 우선순위 상향
- UI는 있으나 API가 없는 경우 → 우선순위 하향 또는 API 개발 필요
- 의존성 관계 고려 (예: 댓글 기능은 포스트 상세 페이지 필요)

### 5단계: 다음 단계 제안

현재 상태를 바탕으로 다음에 구현해야 할 항목들을 제안합니다:

#### 즉시 구현 필요 (1-2주)

- 가장 높은 우선순위 항목
- API가 완성되어 있어 바로 개발 가능한 항목
- 다른 기능의 기반이 되는 항목

#### 보완 작업 (2-3주)

- 부분적으로 구현된 항목 완성
- 통계 및 분석 기능
- 사용자 경험 개선

#### 고급 기능 (나중에)

- 선택적 기능
- 통계 및 분석 고급 기능
- 구독 관리 고급 기능

## 분석 항목 상세

### 관리자 기능 분석

`admin-priority-analysis.md`를 기반으로 다음을 확인합니다:

1. **관리자 대시보드**

   - 페이지 라우트 존재 여부
   - 컴포넌트 구현 여부
   - API 엔드포인트 존재 여부

2. **관리자 사용자 관리**

   - 목록 컴포넌트 구현 여부
   - 상세 컴포넌트 구현 여부
   - 폼 컴포넌트 구현 여부
   - 페이지 라우트 존재 여부
   - API 엔드포인트 완성도

3. **관리자 카테고리/태그/포스트 관리**

   - 각 기능별 컴포넌트 구현 상태
   - 페이지 라우트 연결 상태
   - API 엔드포인트 활용 상태

4. **관리자 댓글 관리**

   - 컴포넌트 구현 여부
   - 일반 댓글 API 활용 여부

5. **통계 기능**
   - 각 통계 컴포넌트 구현 여부
   - 통계 API 엔드포인트 활용 여부

### UI 컴포넌트 분석

`ui-components-priority.md`를 기반으로 다음을 확인합니다:

1. **인증 관련 컴포넌트**

   - 로그인, 회원가입 폼
   - 비밀번호 변경
   - 로그아웃
   - 세션 관리

2. **사용자 관련 컴포넌트**

   - 프로필 조회/수정/삭제
   - 구독 설정
   - 계정 생성

3. **포스트 관련 컴포넌트**

   - 포스트 목록
   - 포스트 상세
   - 포스트 북마크
   - 조회/공유 로그

4. **댓글 관련 컴포넌트**

   - 댓글 목록
   - 댓글 상세
   - 댓글 작성/수정/삭제

5. **카테고리/태그 관련 컴포넌트**

   - 목록/상세 컴포넌트
   - 태그 클라우드
   - 구독 기능

6. **관리자 관련 컴포넌트**
   - 대시보드
   - 각 관리 기능 컴포넌트
   - 통계 컴포넌트

### 페이지 라우트 분석

각 컴포넌트에 대응하는 페이지 라우트가 존재하는지 확인합니다:

- 공통 페이지
- 인증 페이지
- 사용자 페이지
- 관리자 페이지

페이지 라우트는 존재하나 내용이 비어있는 경우도 체크합니다.

## 문서 갱신 프로세스

### 갱신 대상 파일

분석 결과는 다음 두 파일에 반영합니다:

1. `md/admin-priority-analysis.md` - 관리자 기능 우선순위 분석
2. `md/ui-components-priority.md` - UI 컴포넌트 개발 우선순위

### 갱신 내용

#### admin-priority-analysis.md 갱신 항목

1. **요약 섹션**

   - 현재 전체 상태 요약
   - 주요 변경사항

2. **가장 먼저 해결해야 할 항목**

   - 우선순위 재평가 결과
   - 현재 상태 업데이트
   - 필요한 작업 목록 갱신

3. **우선순위별 작업 계획**

   - Phase별 작업 목록 업데이트
   - 예상 소요 시간 조정

4. **API 엔드포인트 현황**

   - 완전히 구현된 API 목록
   - 부분적으로 구현된 API 목록
   - 구독 관리 API 현황

5. **결론**
   - 가장 먼저 해결해야 할 것
   - 다음으로 해결할 것
   - 이유 및 근거

#### ui-components-priority.md 갱신 항목

1. **요약 섹션**

   - 현재 개발 현황 통계 업데이트
   - 전체 진행률 재계산
   - 주요 발견사항 갱신

2. **현재 존재하는 페이지 및 컴포넌트**

   - 페이지 구조 현황 업데이트
   - 컴포넌트 목록 현황 업데이트
   - 상태 표시 (✅, ⚠️, ❌) 갱신

3. **개발 진행 현황**

   - 전체 통계 재계산
   - 우선순위별 진행률 재계산
   - 주요 발견사항 갱신

4. **필요한 컴포넌트 목록**

   - 각 컴포넌트의 상태 업데이트
   - 우선순위 재평가
   - 엔드포인트 연결 상태 확인

5. **개발 단계별 제안**
   - 각 단계별 진행률 업데이트
   - 현재 상태 설명 갱신

### 갱신 시 주의사항

1. **정확성**: 실제 파일 존재 여부를 정확히 확인한 후 갱신
2. **일관성**: 상태 표시 기호 (✅, ⚠️, ❌)를 일관되게 사용
3. **상세성**: 각 항목의 현재 상태를 구체적으로 기술
4. **우선순위**: 객관적인 기준에 따라 우선순위 재평가
5. **날짜**: 분석 날짜 또는 기준 시점 명시 (선택사항)
